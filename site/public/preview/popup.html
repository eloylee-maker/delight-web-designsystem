Popup

Captures attention and prompts a single, well-defined action — registration, confirmation, or critical notice. Because popups appear above page content and interrupt flow, they're used sparingly. Layout, content composition, and allowed sizes below; anything outside this set requires design review.

Specs
Default size
572 px wide · min-height 480 · max-height 680
Surface
White (#FFFFFF), radius 12 px, drop shadow 01 (0 4px 14px 3px rgba(33,33,33,0.10)). Centered over a dimmed page scrim.
Title
Serrif Medium 20 / 1.28, color WGray900. Close icon (✕) top-right, 20×20 hit target.
Form fields
Required marker is the red asterisk (*). Inputs are 1 px --border, radius 8 px, placeholder text in --text-disabled.
Actions
Two-up footer — Close (Secondary, left) → Submit (Primary, right). Always Secondary → Primary.
Confirmation
After Submit, the popup dismisses and a toast confirms — dark #18140F pill, bottom-center, auto-dismiss.
Mobile
Same modal pattern, 88% viewport width, padding scales down. Actions remain Close → Submit.

When to use

Use a popup to capture attention for a single, well-defined action — event registration, gated content, critical confirmation, or a focused form that interrupts the page intentionally.

Don't use a popup for general navigation, ambient promotion, or anything that can live inline on the page. Keep messaging clear and concise — the key information and primary CTA must be understood at a glance.
