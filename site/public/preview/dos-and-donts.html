Do's & Don'ts

The non-negotiable rules of the delight.ai design system. Every page must pass these before shipping. Audited automatically by the design-fidelity-check skill.

1
Background is always pure white
delight.ai pages use #FFFFFF as the page background. Never gradient, never tinted. Use --bg-subtle only for section breaks, footer, and small surfaces.
DO #FFFFFF body DON'T tinted/gradient hero bg
2
Never use Sendbird Purple on delight.ai
#7B39FE is a Sendbird-only accent. delight.ai's accent is Lime #F2FF66, used sparingly.
DO Lime accent (sparingly) DON'T Sendbird Purple
3
Lime is for the CTA banner only
The lime button (.btn-lime) lives only inside .cta-banner on dark background. Body CTAs are black (.btn-primary).
DO black CTA in body DO lime CTA on dark banner DON'T lime CTA on white
4
GNB is position: fixed + spacer
Always use .gnb-wrap with position: fixed, followed immediately by .gnb-spacer (height 72px). Sticky causes layout shift on scroll.
DO fixed wrap + spacer DON'T position: sticky
5
Footer is bg-subtle, not white
Footer background is var(--bg-subtle) (#F7F5F0). The brand column is 284px wide with 116px right margin; link grid is repeat(4, 240px) gap 64px.
DO bg-subtle footer DON'T white footer
6
All type carries letter-spacing: -0.02em
Every text class in this system bakes in -0.02em tracking. Don't override.
7
EN uses Serrif + Helvetica Now Text. KO uses Pretendard.
Headlines: Serrif (EN) / Pretendard 600 (KO). Body: Helvetica Now Text (EN) / Pretendard 400 (KO). Gellix is for badge labels only.
DON'T use Inter, Roboto, system-ui
8
Max content width 1552px
Use the container pattern: width: min(calc(100% - 32px), var(--max-w)). Default offset 184px on PC, 16px on mobile.
9
No emoji, no marketing fluff, no invented colors
Voice is confident, declarative, lowercase-comfortable. Avoid emojis (unless brand-explicit) and "powered by AI" filler. Never invent a color outside the token set — extend tokens instead.
